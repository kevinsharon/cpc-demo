A GitHub Primer

Created for: Central Park Conservancy

Presented: June 20, 2017

What you will be able to do after today:

  • Understand terms like: repository, push, pull, commit, and more!
  • Create and use a repository
  • Start and manage a new branch
  • Make changes to a file and push them to GitHub as commits
  • Open and merge a pull request

Getting Started

  1. Go to github.com and sign up for an account
  2. Go to desktop.github.com and download the GitHub desktop app
  3. Install the app, open it and sign in

Create a Repository

  1. Select File > New Repository
  2. Name it cpcDemoJune2017
  3. Select Initialize this repository with a README

Now you have a code repository (or "repo") to work with. This comes with a default branch called master. We're going to add an exiting file to this branch and make some changes to it

Committing

  1. Open index.html
  2. Change text and save file
  3. In GitHub app view your changes, add a summary and commit to master
  4. Select Publish

Your changes are not committed to GitHub and the master branch now has the latest code. Others working on the branch should now Fetch origin to get the latest code.

Branches

  1. Select Branch > New Branch
  2. Name the new Branch production
  3. Select production under current branch
  4. Open the index.html, make a change and save it, and commit your changes to the production branch

Multiple developers can work with the same repo, within their own branch or on a shared branch. If you're developing specific featues on the site you can create a branch just for working in that new feature. The other developers won't see that feature unless they work in that branch or you branch gets merged into the master branch.

Merging

  1. Select master as the current branch
  2. Select Branch > Merge into current branch
  3. Select production and merge into master
  4. Select Push to complete the merge

Local Development

GitHub

  1. Go to the repo on GitHub and select Clone or Download. Click the copy icon and get the link to the repo
  2. Open the GitHub App and select File > Clone Repository
  3. Select a local folder to maintain your site and add the link you copied from GitHub

CodeKit

  1. Download CodeKit
  2. Open CodeKit select File > New Project
  3. Select the parent folder of the project and select Add

Download the Database

  1. Log in to the Craft control panel from the staging site
  2. Go to the settings page and download the Backup Database
  3. Save this file in an easy to find location
  4. Go to the local file and unzip it

Sequel Pro

  1. Download Sequel Pro
  2. In the main window select Socket
  3. For Name enter Localdev and for Username and Password, enter Root, then select Connect
  4. Select Database > Add Database
  5. For Database Name enter cpc_projects
  6. Select File > Import and select the .sql file

MAMP/Running the site locally

  1. Download MAMP
  2. Click on the gear icon Web Server > Document Root (folder icon) and select the Public folder inside of the repo you just cloned
  3. Select Start Servers
  4. Select Open WebStart page
